# 2026-07-12 — Session 1.3: Attachments & the file-first door

## What was done

- **Vault card** (`web/src/features/patterns/components/AttachmentsCard.tsx`): owner-only on
  pattern detail — lock + "Only you can ever see these."; file rows with token-gated View links;
  plain-language delete confirm; dashed add-file tile; typed-pattern section (TipTap via
  `NotesEditor`, which gained an optional `ariaLabel` prop); SPEC §8 thumbnail backfill when the
  pattern has no cover.
- **Protected files** (`web/src/features/shared/protectedFiles.ts`): `useFileToken` prefetches
  `pb.files.getToken()` (60 s refresh) so View renders as a plain `<a href>` — no
  `window.open`-after-await for iOS popup heuristics to block.
- **PDF pipeline** (`web/src/features/shared/pdfThumbnail.ts`): pdfjs page-1 → canvas → WebP
  (JPEG fallback, imagePipeline's trust-but-verify rule); dynamic import keeps pdfjs out of the
  main bundle; worker self-hosted via Vite `?url`.
- **Quick-add sheet** (`QuickAddSheet.tsx` + `Dock.tsx`): dock ➕ now opens the two-door sheet
  (Add a file · Type it in; link door lands in 3.2). File door processes the pick in-sheet
  (PDF soft-fails thumbnail; images run the §8 pipeline, hard-fail) and hands off via
  `pendingAttachmentImport.ts` (one-shot, StrictMode-safe).
- **Form wiring** (`PatternForm.tsx`, `PatternFormPage.tsx`): pre-filled title/thumbnail + 📎
  caption; create-mode saves pattern → attachment; attachment failure toasts loudly and lands on
  detail (the card is the retry path).
- **Data layer**: `AttachmentRecord` (schema.ts), `attachmentQueries.ts`, `attachmentMutations.ts`
  (one record per file; 30 MB client cap).
- **rules-check.mjs**: `expectFileOk`/`expectFileDenied` + real-file fixture (inline PDF bytes) —
  no token → denied, B's token → denied, A's token → 200; cleanup rides the existing cascade.
- **Bugfix** (`package.json`): `seed`/`verify:rules` used `--env-file-if-missing`, a flag that has
  never existed in Node — the real v22.9+ flag is `--env-file-if-exists`. Latent since 1.1: every
  green run had gone through `verify-fresh.sh`, which sources `.env` itself.
