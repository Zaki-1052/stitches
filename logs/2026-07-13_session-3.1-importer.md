# 2026-07-13 — Session 3.1: Importer sidecar

## What was done

- **Importer workspace** (`importer/`): Fastify 5.10.0 service implementing SPEC §10 —
  `src/{server,app,config,errors,auth}.ts`, `src/net/{parseTargetUrl,ipGuard,guardedFetch}.ts`,
  `src/extract/normalize.ts`, `src/routes/{extract,image}.ts`; tsconfig (bundler mode,
  mirrors web) and `build.mjs` (esbuild → single `dist/server.mjs` with the required
  createRequire banner; 5.6 MB, warning-free).
- **SSRF pipeline**: scheme allow-list + credential strip → DNS resolve with allow-only-
  `unicast` (ipaddr.js; IPv4-mapped unwrapped so `[::ffff:127.0.0.1]` blocks as loopback) →
  undici fetch with a per-hop Agent whose `connect.lookup` returns only the validated
  addresses (checked IP ≡ dialed IP — rebinding closed) · manual redirects ≤3 with the full
  guard re-run per hop (scheme too) · one 8 s budget across hops · caps enforced while
  streaming (3 MB html / 10 MB image) · content-type required.
- **Auth**: caller's PB token forwarded to **POST** auth-refresh (SPEC's "GET" is a typo),
  5-min positive-only cache, no admin creds; PB-unreachable → 401 + `pb_unreachable` event.
- **Rate limit**: @fastify/rate-limit 10.3.0 at `onRequest` after the auth hook, 20/min
  keyed by user id (README's `preHandler` would exempt malformed-body 400s).
- **Contract**: `POST /import/extract` → OGS 6.12.0 `{ html }` → normalized JSON (relative
  image/canonical resolved against the final post-redirect URL); `POST /import/image` →
  buffered image/* passthrough. Errors: 400/401/403/422/429 via one `HttpError` map.
- **Wiring**: `scripts/dev.sh` now starts the importer (tsx watch, exec'd so the trap kills
  it); root `verify:importer` script; web version → 0.3.1 (Vite proxy already existed).
- **Regression**: `scripts/verify/importer-check.mjs` — 401/400 cases, six offline 403
  block cases, `[net]` cases (localtest.me, httpbin redirect-ending-private, PNG→422,
  >3 MB WHATWG page→422, real-page extract + contract-shape + og:image through
  /import/image), and a 21-request burst as user 2 → 429.
